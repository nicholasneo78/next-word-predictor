Brief Description on Next Word Predictor

Introduction

A capstone project from the Data Science Specialisation by Johns Hopkins University and SwiftKey

References can be found in the "References" tab

Goal

The goal of this project is to take in an input from the user which can be a word, a phrase or an incomplete sentence. The algorithm will then use the inputs given and predict what the next word should be based on the SwiftKey datasets. The datasets consist of the twitter, blogs and news dataset in 4 languages. For simplicity, this application will only take in English inputs and only produces an English word prediction as output.

What kind of project is this?

This project incorporates Text Mining and Natural Language Processing (NLP). The large text datasets from the 3 sources are scraped into .txt files which will then be further processed with data cleaning. The cleaned datasets are then ready to do predictions.

Pipeline of this project

These are the rough steps taken for this project:

  • Web Scraping to create the raw datasets

  • Process the raw datasets by cleaning the dataset

  • Do exploratory data analysis to get insights from the datasets

  • Do ngram analysis and determine which combination of words happen more frequently than the other combinations

  • Devise an algorithm for the prediction

  • Set up the web application using Shiny

  • Deploy the application on shinyapps.io

Fun Fact

The default sentence in this web app, "the quick brown fox jumps over the lazy (dog)" has all 26 English alphabets in it. This sentence had been used widely in the Natural Language Processing Community!
